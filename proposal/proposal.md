## 專題動機

在巨量資料與雲端運算課程中，除了學習資料處理方法，也需要理解如何把分析結果做成可以展示與操作的系統。海洋環境資料包含海溫、潮位、浪高、風速與鹽度等欄位，資料型態清楚，而且能呈現時間與測站之間的變化，因此適合作為期末專題題材。

原始 CSV 資料雖然適合儲存，但若只用表格觀看，不容易快速看出測站差異或時間趨勢。本專題希望透過 Python 資料分析工具，把資料清洗、統計摘要、分組分析與圖表視覺化整合成一個可以操作的儀表板，並練習使用 Docker 進行容器化部署。

## 問題背景

海洋觀測資料通常會依照日期與測站持續累積。當資料量增加後，使用者可能會遇到以下問題：

- 欄位型態不一致，例如日期或數值欄位需要轉換。
- 原始表格不容易直接看出趨勢與差異。
- 不同測站之間的比較需要額外整理與統計。
- Demo 或部署時，環境套件版本可能導致系統不容易重現。

因此，本專題將資料處理流程包裝成一個互動式 Web 系統，讓使用者可以用瀏覽器完成資料預覽、清洗、分析與視覺化。

## 系統架構

```text
sample CSV / uploaded CSV
→ data_loader.py 讀取資料
→ clean_data.py 清洗與轉換資料
→ analyze_data.py 統計分析
→ Matplotlib 產生圖表
→ gradio_app.py 顯示互動式儀表板
→ Dockerfile 容器化部署
```

系統主要分為資料讀取、資料清洗、統計分析、視覺化、Web 介面與 Docker 部署六個部分。使用者進入 Gradio 介面後，可以選擇使用範例資料或上傳 CSV，再選擇分析欄位、測站與圖表類型，系統會即時回傳資料表與圖表結果。

## 系統功能規劃

- 使用內建 sample CSV 資料。
- 上傳使用者自訂 CSV 資料。
- 檢查必要欄位是否存在。
- 顯示原始資料前 10 列。
- 執行資料清洗並顯示清洗後資料。
- 產生整體統計摘要。
- 依測站進行分組統計。
- 找出指定欄位的最大值與最小值。
- 選擇分析欄位、測站與圖表類型。
- 產生趨勢折線圖、測站平均長條圖與月份平均折線圖。
- 使用 Docker 啟動後透過 `http://localhost:7860` 操作。
